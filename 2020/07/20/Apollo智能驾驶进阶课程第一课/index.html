自动驾驶(Autonomous Driving)

为什么需要自动驾驶？

93%

是人为原因所造成的交通事故比例。相较于人类，无人驾驶具有以下优势：

  • 处理信息速度快。人类在驾驶车辆时处理信息的速度为30–100条/min，而无人驾驶车辆可以达到3,000条/min
  • 感知环境角度广。无人驾驶车辆车身遍布传感器，可以同时“看到”前后左右的环境

因此，无人驾驶在发展成熟的时候，一定比人类驾驶更加安全。

道路更加顺畅

无人驾驶可以使道路更加通畅。具体体现在：

  • 无人驾驶可以减少道路上的交通事故，从而减少道路拥堵

  • 无人驾驶的反应时间远小于人类驾驶员。

    例如，当人类驾驶员驾驶机动车通过十字路口时，可能需要减速观察路口情况，确认安全后再恢复原速通过；而无人驾驶车辆在经过计算得知（这个过程十分短暂）路口在通过时不可能会变成红灯，也不会有车辆通过时，就会全速通过路口。

无人驾驶分级

###Level 1: Adaptive Cruise Control

无人驾驶系统可以纵向控制车辆，根据周围情况作出加速、减速等行为

Level 2: Lane Keeping

无人驾驶系统还可以横向操控车辆，控制车辆沿车道线（可能是弯曲的车道线）行驶。此时，人类还是驾驶行为的主体

e.g. Autopilot from Tesla

Level 3: Lane Change

无人驾驶驾驶系统在特定情况下成为驾驶行为的主体

e.g. 2018 Aodi A3

Level 2 与 Level 3 之间最重要的区别在于权责的鉴定。在Level 2阶段，人类是驾驶行为的主体。例如，Tesla在其Autopilot功能的介绍页面中就如下提醒用户：

Current Autopilot features require active driver supervision and do not make the vehicle autonomous.

当前的Autopilot功能需要驾驶员时刻进行监管，请勿使车辆处于无人驾驶状态。

Level 4 & 5: Driverless

无人驾驶系统在绝大多数情况下作为驾驶行为的主体。其中，在 Level 5 阶段，车辆内部将不存在方向盘、油门踏板、刹车踏板等操作装置，人类将完全不介入驾驶行为

Apollo开源模块讲解

Overview

民用技术领域技术最复杂，技术含量最高的行业之一

  1. By-wire System: 无人驾驶汽车采用“线控”的方式控制车辆，车内的计算机通过电机驱动车轮。
  2. 强大的处理器：无人驾驶车辆搭载的激光雷达(LiDAR)每天产生的数据在TB级别，因此，需要性能强劲的处理器对这些数据进行处理。
  3. Cluster: 无人驾驶车辆需要实时和云端进行沟通，报告自己的位置，云端需要实时对无人驾驶车辆的位置进行监控跟踪。

HD Maps

为什么无人驾驶需要高精地图？

  1. 使无人驾驶车辆具有预判能力。即使无人驾驶车辆并没有“看到”路面上的减速标志，车辆仍然会提前减速；
  2. 减少无人驾驶车辆的计算量。无人驾驶车辆知道特定地标（如红绿灯）的精确坐标后，就不需要在摄像头的整个视野内检索，只需关注ROI中的内容。
  3. Road Map 和 Intensive Map。Road Map 标注出了车道与车道之间的连接关系，Intensive Map 则主要关注道路周围的静态障碍物。当 Intensive Map 建立好后，无人驾驶车辆在路面上行驶的时候就会忽略这些障碍物。

Localization

有了高精度地图后，最重要的一步就是知道车辆在高精度地图上的什么位置，亦即定位(Localization)。

GPS(Global Positioning System)

原理

通过接收GPS卫星发出的电磁波信号传播的时间，计算卫星与接收者之间的距离，从而确定接收者在地球上的位置。理论上4颗GPS卫星就可以确定地球上的任一位置。

缺陷

  1. GPS发出的电磁信号易受到干扰。例如，当电磁信号穿过电离层时，信号会发生衰减，传播速度也会变慢。再比如，电磁信号在会玻璃幕墙上发生反射，导致信号传播时间变长。
  2. GPS系统计算得出的位置可能产生“瞬移“现象。由于GPS系统计算的位置是时间独立的，加之每一次计算都存在误差，故而导致通过GPS系统计算出的位置信息可能会呈现出”瞬移“的现象。

因此，GPS定位系统的精度只能到达m级。

RTK(Real-Time Kinematic)

原理

建立静态基站接受GPS信号，将接受到的信号与车辆接受到的GPS信号进行差分运算，一定程度上减少误差。RTK可以将定位的误差缩小到10cm级。

缺陷

RTK基站所能覆盖的有效范围有限，一般来说只能保证半径为10miles(≈16km)内有效。对于长距离道路（如国道）来说，完全采用GPS+RTK的定位方案需要建立许多基站，不具备可行性。

IMU(Inertia Measurement Unit)

原理

通过测量车辆的朝向、速度、加速度等运动学物理量，加上初始条件下通过某种定位系统（通常是GPS）给出的位置信息，就可以计算出车辆在给定时刻的位置。这样就解决了GPS测量车辆位置时呈现的“瞬移”问题。

缺陷

由于IMU是通过计算得知车辆的位置，故计算过程中得到的误差会产生累计。因此，经常将GPS和IMU结合作为定位方案。GPS和IMU在实际定位过程中相互校准，减少误差。

然而，当车辆行驶至城市谷地、长距离隧道时，车辆将无法接收到GPS信号。因此，GPS+IMU的方案也不能满足无人驾驶的需要。

几何定位(Geometry Localization)

原理

几何定位也称作相对定位，通过测量与已知精确位置的特征的距离来计算车辆的位置。GPS也是相对定位的一种（通过测量与卫星之间的距离）。实际应用中，通常选择使用LiDAR测量距离，可以达到很高的定位精度。

有关LiDAR

优点

  • 测量精度高
  • 可以测量深度（距离信息）

缺点

  • 对环境要求高。由于 LiDAR 采用纳米级波长的激光束，因此很容易被雾霾中的分子
